Комментарии по реализация ТЗ:
1) Главная страница с доской объявлений: http://127.0.0.1:8000/ads/

2) Регистрация по e-mail с кодом подтверждения регистрации:
Пользовательский интерфейс:
В случае если на сайт заходит незарегистрированный/незалогиненный юзер, форма входа (с опцией регистрации)
открывается при нажатии на ссылку Your Account в верхнем правом углу страницы.

Как реализовано: см. пакет pers_acc
В pers_acc.models описана модель OneTimePassword для генерации одноразовых паролей. При генерации пароля
он по сигналу отправляется на указанную почту (pers_acc.signals). При вводе пароля в форму подтверждения создается
новый объект User, принимающий часть информации из OneTimePassword (имя юзера, емэйл, пароль), после чего
сгенерированный объект OneTimePassword стирается.

3) Приватная страница с откликами объявления/фильтрами
Пользовательский интерфейс: страница доступна при переходе по ссылке Your Account для залогиненного юзера.
Удалить/принять отклик: кнопки DELETE/ACCEPT под каждым откликом.

Как реализовано:  фильтры - см. advert_table.filters; для удаления - дженерик DeleteView. При принятии отклика
меняется булево поле "accepted" (нужно для отправки письма по сигналу).

4) Создание и редактирование объявлений:
Пользовательский интерфейс:
Предварительно пользователь с личной страницы нажатием на кнопку START открывает страницу для ввода имени автора (может
не совпадать с юзернеймом для входа на сайт).
После для автора создание объявлений возможно с главной страницы или с личной страницы (кнопка ADD YOUR AD).
Редактирование и удаление объявлений доступно автору при переходе на вью конкретного объявления (ссылка Read & comment!).
Другие пользователи видеть кнопки EDIT/DELETE не будут.

Как реализовано: см. пакет advert_table.

5) Отклики на объявления других пользователей, состоящие из простого текста:
Пользовательский интерфейс: возможность комментировать доступна при переходе на вью конкретного объявления (ссылка Read & comment!,
затем - ссылка COMMENT).

Как реализовано: через "готовую" форму CommentForm(ModelForm) (см. advert_table.forms).

6) Картинки, встроенные видео и другой контент в объявлении:
Как реализовано: Для добавления другого (помимо текста) типа контента использован виджет CKEditor.

7)Отправка сообщений: автору объявления - при отправке отклика на объявление; пользователю, оставившему отклик - при принятии отклика.
Как реализовано: через сигналы о создании/изменении соответствующей модели (см. advert_table.signals)

8) Отправка новостных рассылок:
Пользовательский интерфейс: автоматическая еженедельная рассылка новостей за неделю (в 00.00, понедельник).
Как реализовано: через apscheduler (см. advert_table.management).
